# tontonTetoris サウンド素材 制作依頼書

落ちものパズル『tontonTetoris』向けの **BGM 1曲＋SE 2種** の制作をお願いします。
本書だけで制作に着手できるよう、用途・質感・技術仕様をまとめています。不明点は遠慮なくご質問ください。

---

## 0. ゲーム概要（音の土台になる前提）

- ジャンル：物理ベースの落ちものパズル（ぷよぷよ風＝同じ色のブロックが隣接して一定数集まると消える）。
- 特徴：ブロックが**実際の重力で落ちて積み上がる**。同じ色のブロックは触れると**「吸着結合（ドッキング）」**して塊になる。条件を満たすと消え、**連鎖**するほど画面がスロー＋フラッシュ＋衝撃波で派手に盛り上がる。
- 配信：itch.io 等でのブラウザ／PC配信を想定。
- **世界観・トーン：レトロ／チップチューン（8bit〜16bit のアーケード感）。**
  - 基本はファミコン／ゲームボーイ系のチップ波形（矩形波・三角波・ノイズ）を土台に。
  - 完全な実機制約（"純チップ"）／現代的な処理を足した "fakebit" のどちらでも可。**迷ったら "純チップ寄り＋軽い味付け" を基準**にしてください。
  - 参考にしてほしい音の質感：『ぷよぷよ』『テトリス（GB/アーケード）』『パズルボブル』系のレトロパズル。

## 1. BGM（本編ループ） — 1種

| 項目 | 内容 |
|---|---|
| **ファイル名** | `bgm_main_loop.ogg` |
| **用途・再生タイミング** | タイトルではなく、**モードを選んで実際にブロックを操作している本編中**に再生。ゲーム中はずっとループで鳴り続ける（ゲームオーバー／タイトルに戻ると停止）。 |
| **雰囲気** | 中庸テンポでノリが良く、邪魔をせず気持ちよく続くループ。集中と高揚のバランス型。「ずっと聴いていても疲れない」ことを最優先。 |
| **テンポ目安** | 120〜140 BPM 程度。 |
| **編成イメージ** | チップチューン定番の編成：リードに矩形波（パルス）、ベースに三角波、リズムにノイズチャンネル。明るめのメジャー〜少しおどけた雰囲気。 |
| **構成・尺** | **シームレスにループする 30〜60 秒程度**。イントロ→メインで1ループ完結し、終端が先頭へ綺麗につながること（無音の継ぎ目・プツッというノイズ厳禁）。 |
| **避けたいもの** | 主張が強すぎる長尺メロや、急な静寂・急なドラマ展開（ループの不自然さや飽きの原因になるため）。 |
| **参考トーン** | 『ぷよぷよ』通常戦、レトロパズルのフィールドBGM。 |

> ※ループはファイル自体が完全ループになるよう作ってください（エンジン側はファイル末尾→先頭で繋ぐ前提。ループ設定が無くても繰り返す保険は実装済みですが、**継ぎ目レス前提**でお願いします）。

## 2. SE：ドッキング（吸着結合）音 — 1種

| 項目 | 内容 |
|---|---|
| **ファイル名** | `se_dock.wav` |
| **用途・再生タイミング** | プレイヤーが操作するブロックと、同色ブロックが**くっついて1つの塊になった瞬間**に1回再生。プレイ中に**高頻度で連打的に鳴る**（操作のフィードバック音）。 |
| **質感** | **「カチッ／パチッ」と短く確定的に吸着する手応え**。マグネットが吸い付くような、気持ちよく「決まった」感。 |
| **音のキャラ** | チップ波形による短いブリップ／プラック。アタックは速く、歯切れよく。ほんの少しだけ「吸い込む」ニュアンス（ピッチが一瞬上にスッと上がる等）があると◎。 |
| **長さ** | **80〜150 ms 程度**（とにかく短く・軽快に）。 |
| **重要要件** | 高頻度で連打されるので、**何度連続で鳴っても耳に痛くない／うるさくない**こと。低〜中音域中心で、刺さる高域は控えめに。 |
| **避けたいもの** | 長い余韻、リバーブたっぷり、低音ドスン系（結合は"軽い確定音"であって衝撃音ではない）。 |

## 3. SE：ブロック破壊（消去）音 — 1種 ★ピッチ可変仕様あり

| 項目 | 内容 |
|---|---|
| **ファイル名** | `se_break.wav` |
| **用途・再生タイミング** | ブロックが**消える（破壊される）瞬間**に1回再生。連鎖の各段でそのつど鳴る。 |
| **質感** | **「パリーン／パンッ」と弾ける・破裂する爽快感**。連鎖の盛り上がり（スロー＋画面フラッシュ＋衝撃波）と噛み合う、抜けの良いヒット音。 |
| **音のキャラ** | チップチューンの**ノイズチャンネル主体の破裂音**（8bit爆発／ヒットの質感）＋短いトーン成分でアクセント。「ドンッ」より「パンッ／パリッ」の弾ける方向。 |
| **長さ** | **150〜250 ms 程度**（後述のピッチ仕様のため、短めに作ること）。 |

### ★連鎖ピッチ仕様（最重要・必読）

破壊音は**連鎖数が増えるごとにゲーム側でピッチ（再生速度）を自動的に上げます**。素材は**1連鎖目＝原音**として作ってください。重ねがけや音違いを焼き込む必要はありません（**1ファイルでOK**）。

ゲーム側の倍率（初期設定）：

| 連鎖数 | 再生ピッチ倍率 |
|---|---|
| 1連鎖 | ×1.00（原音） |
| 2連鎖 | ×1.15 |
| 3連鎖 | ×1.30 |
| 4連鎖 | ×1.45 |
| 5連鎖以上 | ×1.60（頭打ち） |

制作上の注意：

- **`se_break.wav` を原音とし、再生時に最大 ×1.60 まで速く・高くなっても破綻しない**ように作ってください（Godotの`pitch_scale`は再生速度ごと上がるため、**高ピッチ時は音が短く・甲高くなります**）。
- 納品前に、**原音を 1.0／1.15／1.3／1.45／1.6倍速で試聴**し、最高速でも「ピーキー過ぎず気持ちよく弾ける」ことを確認してください。
- そのため**原音は欲張って長くせず、低めの基音**で作ると高ピッチ側が破綻しにくいです。
- 高頻度で連続するので、こちらも**連打で耳に痛くならない**バランスを優先。

## 4. 共通の技術仕様（納品フォーマット）

| 項目 | BGM | SE（2種共通） |
|---|---|---|
| ファイル形式 | **OGG Vorbis（.ogg）** | **WAV（.wav, PCM 非圧縮）** |
| サンプルレート | 44.1 kHz | 44.1 kHz |
| ビット深度 | 16-bit 相当 | 16-bit |
| チャンネル | ステレオ | モノラル推奨（ステレオ可） |
| ラウドネス目安 | 約 **-16 LUFS**（統合） | ピーク **-3 dBFS** 程度に正規化 |
| ヘッドルーム | クリップ（0 dBFS 超）厳禁。-1 dBFS 以下に収める | 同左 |
| ループ | **完全シームレスループ必須** | ループ不要（ワンショット） |

- 音量の最終バランスはゲーム側のボリューム調整（dB）で行うため、**素材は上記基準で正規化**して納品してください。極端に小さい／大きいファイルは避けてください。
- 無音の前後余白（特にSE先頭）は極力詰めてください（発音の遅延になるため）。
- 命名は上記の3ファイル名どおりでお願いします（差し替えがスムーズになります）。

## 6. ゲーム側の組み込み（依頼者メモ／音屋さんは読まなくてOK）

納品ファイルは `res://assets/Sound/` に配置し、Godotエディタで `sound_settings.tres` を開いて以下に割り当てる：

- `BGM > Bgm Stream` ← `bgm_main_loop.ogg`
- `SE - Docking > Dock Stream` ← `se_dock.wav`
- `SE - Block Break > Break Stream` ← `se_break.wav`

ピッチの刻み・上限は `sound_settings.tres` の `break_pitch_step`（既定 0.15）／`break_pitch_max_chain`（既定 5）でいつでも調整可能。
